                                      2. 업로드 준비

                                      업로드 준비

                                      HQ가 배정용 엑셀을 다시 업로드하기 전에 file contract와 메타데이터를 먼저 검토합니다.

                                      REPLACE 정책

                                      업로드한 파일의 최신 내용으로 기존 지원근무자 배정을 덮어씁니다. 재업로드 시 기존 명단은 병합되지 않고 최신 파일 기준으로 교체됩니다.

                                      3. 적용 전 검토

                                      적용 전 검토

                                      이번 phase에서는 workbook validity, revision 상태, scope/month mismatch, 시트 범위만 구조적으로 검토합니다.

                                      업로드 파일 검토를 시작하면 review 구조가 여기에 표시됩니다.

                                      scope별 요청 수, 유효 입력 수, 근무자 셀 파싱 결과가 이 구역에 표시됩니다.

                                      시트명 지점 날짜 구분 항목 입력값 유효/요청 target 상태 검토 상태 사유
